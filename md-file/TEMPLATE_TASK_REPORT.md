## 编写建议

### ✅ 推荐做法

1. **分层清晰**
   - 使用 h2 (##) 为主要章节
   - 使用 h3 (###) 为子章节
   - 使用 h4 (####) 为详细项目

2. **代码示例**
   - 包含真实的代码片段
   - 用 ``` 标记代码块
   - 指定语言 (vue/js/css)

3. **表格使用**
   - 用于对比数据
   - 统计数据
   - 配置清单

4. **检查清单**
   - 使用 [x] 表示完成
   - 使用 [ ] 表示待完成
   - 分类清晰

### ❌ 避免做法

1. 过长的代码块（> 50 行）→ 只保留关键部分
2. 重复说明 → 简洁明了
3. 主观评价 → 基于数据和事实
4. 格式混乱 → 保持 Markdown 规范
